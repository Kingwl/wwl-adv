# 游戏设计文档

> 游戏 UI 信息架构、HUD、升级选择、暂停和结算界面设计见 [`ui_design.md`](./ui_design.md)。
> 本地数值存档设计见 [`save_system_design.md`](./save_system_design.md)。
> 当前角色、武器、被动和构筑结构整理见 [`content_structure.md`](./content_structure.md)。

## 游戏概述

- **名称**: WWL Adventure
- **类型**: 吸血鬼幸存者 like 实时生存 Roguelike
- **视角**: 2D 俯视角
- **目标平台**: Web / Android / iOS
- **单局时长目标**: 5-15 分钟
- **核心乐趣**: 走位生存 + 自动战斗 + 升级构筑

## 核心循环

```text
移动走位 → 自动攻击敌人 → 击杀掉落经验/金币 → 拾取经验升级 →
三选一强化 → 武器解锁/升级/选流派 → 敌人变强 → 循环 → HP归零结算
```

## 当前已实现系统

### 玩家

玩家基础属性由当前选择的 `CharacterData` 决定。默认角色为冒险者。

| 属性 | 默认值 |
|------|--------|
| 初始 HP | 100 |
| 移动速度 | 170 |
| 受击无敌 | 受击后短暂无敌，避免连续扣血 |
| 死亡 | 播放死亡动画后进入结算 |

- 控制方式：键盘 WASD / 方向键 + 触屏虚拟摇杆
- 摄像机跟随玩家
- 头顶血条同步 `GameState.hp_changed`
- 升级时播放脚下光效

### 角色

当前有 4 个默认可选角色，暂不做解锁或购买。主菜单选择后写入本地 profile 的 `selected_character_id`，开新局时读取并应用；不保存战斗中的角色状态。

| 角色 | 初始武器 | HP | 移速 | 被动 |
|------|----------|----|------|------|
| 冒险者 | 基础利刃 | 100 | 170 | 标准数值，无明显短板 |
| 游侠 | 弓箭精通 | 85 | 190 | 弹体类武器冷却缩短 8% |
| 守卫 | 荆棘护甲 | 140 | 145 | 折光甲胄：受伤时减免 12%，并将减免量按距离折射给周围敌人 |
| 炼金术士 | 火焰瓶 | 90 | 160 | 范围类武器范围 +8%，火焰 / 毒雾持续时间 +15% |

### 敌人

当前敌人已通过 `resources/enemies/*.tres` 内容化，共 11 种资源；当前局内时间线使用 7 种普通 / 精英敌人 + 2 波小 Boss + 1 波最终 Boss。

| 敌人 | 定位 | 出现时间 | HP | 速度 | 接触伤害 | 小队 |
|------|------|----------|----|------|----------|------|
| 小鬼 | 群体基线 | 0s | 10 | 58 | 4 | 3 |
| 疾行者 | 高速压迫 | 45s | 8 | 92 | 3 | 2 |
| 邪教射手 | 远程压迫 | 90s | 14 | 48 | 5 | 1 |
| 蛮兵 | 慢速高血量 | 120s | 46 | 38 | 8 | 1 |
| 突袭者 | 短前摇冲刺 | 180s | 18 | 54 | 7 | 1 |
| 精英蛮兵 | 精英肉盾 | 260s | 120 | 42 | 12 | 1 |
| 精英秘法师 | 精英远程 | 360s | 70 | 44 | 10 | 1 |
| 北境督军 | 第一波小 Boss | 300s 事件生成 | 420 | 34 | 10 | 1 |
| 北境霜卫 | 第二波双 Boss | 510s 同时生成 | 360 | 36 | 10 | 1 |
| 北境符文先知 | 第二波双 Boss | 510s 同时生成 | 440 | 42 | 11 | 1 |
| 冰冠霸主 | 最终 Boss | 720s 事件生成 | 760 | 38 | 13 | 1 |

- 行为：追踪型持续接近玩家；突袭者会按冷却进入前摇、冲刺、恢复三个阶段；远程型会保持距离并向玩家发射弹体
- 精英：通过更高 HP / 伤害 / 掉落、更大体型和颜色区分，暂不引入复杂技能
- Boss：当前时间线为 5:00 北境督军、8:30 同时生成北境霜卫 + 北境符文先知、12:00 冰冠霸主；Boss 均不进入普通权重池，使用专属 6 帧 Boss 条带和 4 帧弹体条带。前两波小 Boss 击败后继续游戏，只有带 `final_boss` 标签的冰冠霸主死亡才胜利结算。
- 状态：通过 `StatusEffect` 支持 `slow` 和 `stun`，并预留刷新策略、堆叠和 tick 伤害
- 生成：从玩家周围可视区外的视口半对角线 + 80~300 像素环形区域随机生成，按 `spawn_weight`、时间窗口和 `pack_size` 选择；同一小队内成员按约 72~160 像素的环形偏移分散落点，避免 3 只怪重叠生成
- 难度曲线：生成间隔随时间缩短，但不低于 0.55s；HP / 伤害按 210 秒周期线性成长，速度按 360 秒周期成长并封顶到 1.55 倍；存活敌人上限 150；510s 后进入后段权重，降低小鬼占比并提高突袭、远程和精英出现率
- 奖励曲线：敌人 EXP 会按难度成长获得部分补偿，金币获得较弱补偿，避免中后期只增加压力不增加成长资源
- 资源化：`EnemyData` 记录属性、体型、行为、专属动画条带、生成权重、出现时间和小队数量
- 美术：7 种敌人均有独立 6 帧 64×64 条带，运行时拆成 walk / hit / death 动画

### 技能总览

当前把“技能”按构筑用途拆成两类：

- **主动技能**：占用武器槽，由武器系统自动触发或持续生效。
- **被动技能**：不占武器槽，提供角色数值、规则或恢复效果。

当前升级池可形成的构筑技能共 **30 个**：**20 个主动技能 + 10 个局内被动强化**。如果把 4 个角色自带被动也纳入技能总数，则当前共有 **34 个技能**：**20 个主动技能 + 14 个被动技能**。

| 类别 | 数量 | 来源 | 槽位 | 内容 |
|------|------|------|------|------|
| 主动技能 | 20 | `resources/weapons/*.tres` + `UpgradeSystem.WEAPON_SCENES` | 武器槽，单局最多 6 个 | 基础利刃、弓箭精通、天雷引、护盾球、荆棘护甲、散弹枪、火焰瓶、冰霜环、圣光棱镜、毒液罐、地雷、激光笔、回旋镖、电磁链、锯片陷阱、火箭背包、旋风斩、投掷斧、冲击波、火花弹 |
| 局内被动强化 | 10 | `scripts/game/upgrade_system.gd` 内置 `PLAYER_STAT` | 强化槽，单局最多 6 种 | 疾风步、生命强化、磁力增幅、生命源泉、强攻、专注、扩张、余烬延续、坚韧、历练 |
| 角色自带被动 | 4 | `resources/characters/*.tres` | 不占单局槽位 | 冒险者：标准数值；游侠：弹体冷却 -8%；守卫：折光甲胄；炼金术士：范围 +8%，火焰 / 毒雾持续 +15% |

| 天雷引 | 范围爆发 | 随机选敌落雷 |

| 火花弹 | 爆炸弹体 | 命中后小范围爆炸 |

每种武器：

- 继承 `WeaponBase`
- 有独立 `.tscn` 场景
- 有 `resources/weapons/*.tres` 数据资源
- 默认等级上限 8
- 20 种武器均支持 3 条流派路径，每条路径通过显式路线标签（强击 / 扩散 / 疾速 / 穿透 / 控制 / 守护 / 持续）表达构筑主轴，并通过 `WeaponPathLevel` 提供数值 bonus 或 `special_tag`
- 当前 `special_tag` 均已接到对应武器脚本，自动化测试覆盖主要路径效果
- 路径数值避免纯 `damage_bonus` 堆叠；伤害路线也需要混入冷却、范围或机制 tag

### 伤害系统

当前伤害已经统一为 `DamageEvent → DamageCalculator → target.apply_damage() → DamageResult`。

- `DamageEvent` 记录伤害来源、目标、武器 ID、伤害类型和命中方式。
- `StatusEffect` 记录状态剩余时间、刷新策略、堆叠、数值和周期伤害载荷；简单状态仍可通过 `DamageEvent.status_id/status_duration/status_value` 传递。
- 已使用的伤害类型包括 `physical`、`fire`、`poison`、`frost`、`lightning`、`holy`、`pure`。
- 已使用的命中方式包括 `melee`、`projectile`、`area`、`dot`、`beam`、`contact`、`reflect`。
- “弹幕”共鸣已接入伤害结算：I 档固定增伤，II 档按命中距离额外增伤，III 档命中后短时击退敌人；精英 / Boss 击退力度降低。
- 旧的 `take_damage(int)` 保持兼容，便于测试和后续逐步迁移。
- 敌人死亡后不再重复吃伤害或重复结算击杀。
- 玩家受伤减免、守卫折光甲胄和死亡结算仍由 `GameState` 统一处理。
- 本地调试护身只在 editor / debug 构建可用，开启后由 `GameState` 拦截玩家受到的伤害，不写入存档；键盘输入 `iddqd` 或触屏连续点击生命条 7 次可切换。

### 战斗统计

当前单局会在 `GameState.run` 里记录轻量战斗复盘数据，不进入本地数值存档：

- `weapon_damage` / `weapon_hits` / `weapon_kills`：按武器 ID 统计本局伤害、命中和击杀。
- `weapon_damage_stats`：结算页使用的武器统计聚合，包含武器显示名。
- `damage_taken_by_source`：按敌人接触、敌方弹体等来源统计玩家受到的伤害。
- `death_reason`：记录最后一次致死伤害来源、伤害值和伤害类型；胜利时结算页显示 Boss 击败结果。
- `upgrade_history`：记录本局选择过的升级、流派、角色强化和选择时等级。
- `build_resonance_rewards`：记录本局已触发的构筑共鸣奖励，当前用于 UI 展示，并驱动已接入的弹幕奖励效果。

这些数据当前用于结算页“战斗复盘”和“升级路线”，后续 Debug Overlay、平衡分析和 Web smoke 流程测试可以复用。

### 升级系统

经验公式：

```text
level <= 20: exp_required = 15 * 1.2^(level - 1)
level > 20:  exp_required = 15 * 1.2^19 * 1.1^(level - 20)
```

当前升级类型：

| 类型 | 说明 |
|------|------|
| 角色强化 | 移速、最大 HP、治疗、拾取范围、全局伤害、全局冷却、全局范围、持续场地持续时间、受伤倍率、经验倍率；单局最多占用 6 个强化槽，同类再次出现时叠加等级，单类最高 5 级 |
| 武器解锁 | 从未持有武器中随机出现 |
| 武器流派 | 武器 1 级且未选流派时出现；选择后立即升到 Lv.2 并获得该流派首个强化 |
| 武器强化 | 已选流派后按路径给出下一级效果；硬编码强化回退仍保留给未来无路径武器 |
| 外部升级 | `DataManager.all_upgrades()` 返回的资源会加入候选池 |

武器路径的数值 bonus 由 `WeaponBase` 按当前等级累计结算；路径升级卡上的 `damage_bonus` / `cooldown_bonus` / `range_bonus` 只用于展示，不再由 `UpgradeSystem` 二次叠加。

升级卡只显示标签和共鸣进度：武器解锁、流派选择、后续武器升级和局内被动都会展示本次贡献与选择后的进度，例如 `近身 +0.25 1.75/2.0` 或 `近身 +1.0 解锁I`。当前路线标签只用于决策表达，不参与 P1 共鸣计分。数值按 12 分钟局最多约 2 个满级武器专精设计，两把同构筑满级专精约可触发 III 档，被动用于补足差距。

当前内置角色强化：

| 名称 | 效果 |
|------|------|
| 疾风步 | 移动速度 +25 |
| 生命强化 | 最大 HP +30，当前 HP +30 |
| 磁力增幅 | 拾取范围 +30 |
| 生命源泉 | 立即恢复 5 HP；之后按 `5s * 全局冷却倍率` 自动恢复生命，治疗量随强化等级提升 |
| 强攻 | 所有武器伤害 +8% |
| 专注 | 所有武器冷却 -6% |
| 扩张 | 所有武器范围 +8% |
| 余烬延续 | 火焰 / 毒雾等持续场地持续时间 +12% |
| 坚韧 | 受到伤害 -8% |
| 历练 | 经验获取 +10% |

单局最多持有 6 把基础武器和 6 种角色强化。升级候选会过滤已满级武器、满级强化、满槽后的新武器 / 新强化，并保证同一轮不会重复出现同一个武器 ID。升级面板支持刷新全部候选卡片，也支持跳过本次选择直接继续游戏。

| 全局伤害倍率 | 最高 1.6x |
| 全局冷却倍率 | 最低 0.6x |

### 存档

- `SaveManager` 通过 `user://save_v1.json` 保存本地单槽数值档案，并保留 `user://save_v1.bak`。
- 当前选择角色保存为 `selected_character_id`；所有角色默认可选，当前版本没有角色解锁字段。
- 获得金币和击杀敌人时即时更新累计金币 / 累计击杀。
- 死亡 / 胜利结算时更新总局数、胜利数、最佳成绩和最近一局摘要。
- 不保存玩家位置、HP、EXP、武器、强化、敌人、弹体、掉落物等战斗状态；刷新或重启后从主菜单重新开局。

### UI

- 主菜单：全局累计金币、角色选择、开始游戏
- HUD：时间、击杀、本局金币、HP、EXP、等级、武器栏、强化栏、1x / 2x 倍速按钮、属性按钮
- 升级选择：三张卡片，显示图标、类型和武器分类
- 暂停菜单：展示当前武器槽位、强化槽位并可返回主菜单
- StatsPanel：查看角色基础状态、本局 / 累计金币、已持有强化和武器详情
- GameOver：展示时间、击杀、本局金币、累计金币、本局武器和强化，并显示武器伤害排行、受伤来源、最后伤害和升级路线
- VirtualJoystick：移动端 / Web 自动显示
