# 游戏 UI 设计文档

> 本文档描述 WWL Adventure 的 UI 信息架构、视觉风格和各界面设计。
> UI 素材需求见 [`art_requirements.md`](./art_requirements.md)，整体玩法目标见 [`game_design.md`](./game_design.md)。
> 注意：本文包含未来设计目标；当前已实现的是 HUD、升级三选一、暂停、结算、属性面板和武器分类标签，武器图鉴、设置、刷新/锁定、组合进化仍属于规划。

## 设计目标

- 战斗中 UI 尽量贴边，不遮挡玩家周围 300px 战斗区域。
- 所有核心状态一眼可读：时间、HP、经验、等级、金币、击杀、武器。
- 升级选择要强化 build 感：玩家能看出“新武器 / 强化 / 进化 / 属性”的区别。
- 移动端可用：按钮够大，底部 UI 避开虚拟摇杆。
- 风格保持 2D 像素街机感，服务战斗信息读取，不做复杂装饰。

## 整体视觉

- 风格：2D 像素风 UI，深色半透明面板 + 明亮像素描边。
- 背景面板：深蓝黑 `#10131f`。
- 描边：灰蓝 `#3a4568`。
- 强调金色：`#f2c14e`。
- HP 红：`#d94b4b`。
- EXP 蓝：`#3aa7ff`。
- 进化紫：`#b56cff`。
- 字体：像素字体优先；没有字体资源时先用 Godot 默认字体，但字号和对齐按像素 UI 处理。
- 面板圆角：4-8px，不使用大圆角卡片风。
- 图标尺寸：优先 16x16 / 32x32 PNG。

## 主菜单

主菜单不是营销页，直接服务“开始一局”。

推荐布局：

```text
┌──────────────────────────────┐
│         WWL Adventure         │
│      生存 Roguelike · v0.1    │
│                              │
│        [ 开始游戏 ]           │
│        [ 武器图鉴 ]           │
│        [ 设置 ]               │
│        [ 退出 ]               │
│                              │
│  当前武器：17 / 17            │
└──────────────────────────────┘
```

按钮：

- `开始游戏`：主按钮，金色描边。
- `武器图鉴`：查看已解锁武器和进化配方。
- `设置`：音量、震动、画面缩放、语言。
- `退出`：桌面端显示，Web / 移动端可隐藏。

背景：

- 使用暗色地面纹理或低对比像素图，不使用纯色大空白。
- 可在底部展示玩家、敌人、金币、经验球小图标，增强游戏感。

## 战斗 HUD

当前 HUD 已覆盖时间、击杀、金币、HP、EXP、等级、武器栏和属性入口；后续优化重点是移动端布局和更清晰的优先级。

推荐布局：

```text
┌────────────────────────────────────────────┐
│  05:23        击杀 128        金币 42       │
│                                            │
│                                            │
│                                            │
│                                            │
│  HP ███████████░░ 82/100                   │
│                                            │
│  武器栏  [剑2][弓1][雷3][盾1][空][空]       │
│  Lv.7  EXP ███████░░░░░░░░ 18/42           │
└────────────────────────────────────────────┘
```

信息优先级：

| 信息 | 位置 | 说明 |
|------|------|------|
| 存活时间 | 顶部居中 | 当前局核心目标 |
| 击杀数 | 顶部偏左 / 中 | 成就感信息 |
| 金币 | 顶部右侧 | 后续商店机制需要常驻 |
| HP | 左下 | 生存信息，红色大条 |
| EXP + 等级 | 底部中 / 右 | 升级进度，蓝色长条 |
| 武器栏 | 底部 HP 和 EXP 之间 | 显示 6 个槽位 |
| 当前进化提示 | 武器栏上方短暂出现 | 例如“利刃已满级，寻找回旋镖” |

武器栏状态：

| 状态 | 表现 |
|------|------|
| 已持有 | 显示武器图标 + 等级角标 |
| 空槽 | 暗色虚线框 |
| 满级 | 金色边框 |
| 可进化 | 紫色闪烁边框 |
| 已进化 | 紫金边框 + 特殊角标 |

## 升级选择 UI

升级选择是最重要的构筑界面，应从普通按钮升级为三张选择卡。

推荐布局：

```text
┌──────────────────────────────────────────────┐
│                 选择升级                     │
│                                              │
│ ┌──────────┐  ┌──────────┐  ┌──────────┐     │
│ │  图标     │  │  图标     │  │  图标     │     │
│ │ 新武器    │  │ 武器强化  │  │ 角色属性  │     │
│ │ 弓箭精通  │  │ 利刃强化  │  │ 疾风步    │     │
│ │ 自动射击  │  │ 伤害 +5   │  │ 移速 +15  │     │
│ │ [弹体]   │  │ Lv.2→3    │  │           │     │
│ └──────────┘  └──────────┘  └──────────┘     │
│                                              │
│        刷新 1 次      当前武器 3 / 6          │
└──────────────────────────────────────────────┘
```

卡片类型视觉：

| 类型 | 边框色 | 说明 |
|------|--------|------|
| 新武器 | 蓝色 | 解锁未持有武器 |
| 武器强化 | 金色 | 提升已持有武器等级 |
| 角色属性 | 绿色 | 提升玩家基础属性 |
| 进化 | 紫色高亮 | 触发组合进化 |
| 稀有事件 | 橙色 | 刷新、锁定、复制等特殊选择 |

每张卡必须显示：

- 图标。
- 类型标签：新武器 / 强化 / 属性 / 进化。
- 名称。
- 一句话效果。
- 标签或等级变化。

示例：

```text
[新武器]
弓箭精通
自动瞄准最近敌人发射箭矢
标签：弹体
```

```text
[进化]
风暴刃轮
基础利刃进化，多道刀轮扩张后回收
需要：回旋镖 Lv.4
```

## 暂停菜单

暂停菜单要服务移动端误触和中途查看 build。

推荐布局：

```text
暂停
存活时间 05:23    击杀 128

当前武器
[剑 Lv.5] [弓 Lv.3] [雷 Lv.2] [空] [空] [空]

[继续游戏]
[设置]
[返回主菜单]
```

设计要求：

- 暂停时展示当前武器栏，不只是两个按钮。
- 设置里只放必要项：音量、震动、画面缩放。
- 返回主菜单需要二次确认。

## 游戏结束界面

结算界面要让玩家知道“这局哪里强”。

推荐布局：

```text
游戏结束

存活时间       12:43
击杀数         426
达到等级       Lv.18
金币获得       73
最高武器       风暴刃轮

本局武器
[风暴刃轮] [腐蚀箭雨] [护盾球 Lv.6] ...

[再来一局]
[返回主菜单]
```

可后续增加：

- 新解锁武器提示。
- 最多伤害武器。
- 局外奖励。

## 移动端适配

- 虚拟摇杆左下，占用 120x120 区域。
- HP 条放左上或左中下方，避免和摇杆重叠。
- EXP 条建议横跨底部中间，但留出左下摇杆空间。
- 升级卡片在窄屏上改为纵向 3 张卡，每张高度约 120。
- 所有按钮高度至少 48px。
- 卡片可点击区域不小于 100px 高。

## 实施优先级

| 优先级 | 内容 | 原因 |
|--------|------|------|
| P0 | HUD 重排、升级选择卡片化 | 直接影响战斗和 build 体验 |
| P1 | 武器栏、进化提示、结算武器展示 | 支撑武器系统蓝图 |
| P2 | 主菜单图鉴入口、暂停菜单 build 查看 | 提升完整度 |
| P3 | 设置、移动端细调、局外解锁展示 | 后续系统成熟后再做 |

## 验收标准

- 战斗中玩家周围核心区域不被 UI 遮挡。
- 升级界面能明确区分新武器、强化、属性、进化。
- 武器栏能展示 6 个槽位、等级、满级和可进化状态。
- 结算界面能展示存活时间、击杀、等级、金币和本局武器。
- 移动端虚拟摇杆不与 HP / EXP / 按钮重叠。
