# 战斗效果分层规范

本文档用于约束伤害、Buff、Debuff、共鸣和反伤/回放等派生效果的归属。目标不是重写系统，而是让后续新增效果时知道应该放在哪一层、能触发哪些后续效果。

## 核心入口

当前战斗入口是：

```text
DamageEvent -> DamageCalculator -> target.apply_damage(event) -> DamageResult
```

| 类型 | 职责 |
|------|------|
| `DamageEvent` | 描述一次伤害尝试：基础数值、来源、目标、武器 ID、伤害类型、命中方式、标签和可附带状态 |
| `DamageCalculator` | 计算伤害倍率，并分发命中后、击杀后副作用 |
| `CombatEffectRules` | 集中声明派生伤害标签，以及这些标签是否跳过构筑倍率、命中后效果和击杀后效果 |
| `DamageResult` | 返回原始伤害、最终伤害、减免、暴击、击杀、过量伤害和状态应用结果 |
| `StatusEffect` | 描述目标身上的持续状态：剩余时间、刷新策略、堆叠、数值和 tick 伤害 |

旧的 `take_damage(int)` 仍保留为测试和外部简单调用的兼容包装；战斗主链应构造 `DamageEvent` 并走 `DamageCalculator.deal_damage()` / `apply_damage(event)`。

## 伤害类型与命中方式

伤害类型用于表达抗性、视觉、统计和未来联动：

| 类型 | 当前用途 |
|------|----------|
| `physical` | 斩击、箭矢、碰撞等默认物理伤害 |
| `fire` | 火焰瓶、火场 |
| `poison` | 毒液罐、毒雾 |
| `frost` | 冰霜环等冰霜伤害 |
| `lightning` | 天雷、电磁链 |
| `holy` | 圣光棱镜 |
| `pure` | 反射、回响等不吃普通减免的派生伤害 |

命中方式用于表达距离、来源和入伤规则：

| 方式 | 当前用途 |
|------|----------|
| `melee` | 近身命中，参与近身距离增伤和近身回放候选 |
| `projectile` | 弹体命中，参与弹幕远距增伤和敌方弹体入伤 |
| `area` | 范围或爆炸命中 |
| `dot` | 持续伤害 tick |
| `beam` | 激光等光束命中 |
| `contact` | 敌人与玩家接触伤害 |
| `reflect` | 反伤、回响、回放等派生伤害 |

新增类型或方式前，先确认它是否真的会改变规则。如果只是视觉不同，优先用现有类型加资源表现，不新增规则枚举。

## 出伤结算顺序

对敌人的伤害结算顺序如下：

1. 武器、弹体、场地、状态 tick 或派生效果构造 `DamageEvent`。
2. `DamageCalculator.calculate()` 计算基础伤害、暴击和出伤倍率。
3. 目标 `apply_damage(event)` 扣血、应用随伤害附带的状态、计算击杀和过量伤害。
4. `DamageCalculator` 按顺序处理命中后/击杀后副作用：
   - 控制共鸣：状态成功应用后触发威胁削弱和静滞充能。
   - 爆发共鸣：击杀后按过量伤害外溢。
   - 场地共鸣：非击杀场地命中叠加压制，满层触发地脉封锁。
   - 近身共鸣：记录最近 5 秒最高近身伤害候选。
   - 弹幕共鸣：命中后击退。
5. `GameState.record_damage_result()` 记录武器伤害、命中和击杀统计。

这个顺序很重要：爆发外溢依赖目标已死亡和 `overkill_amount`，场地压制只对存活目标叠层，统计记录的是最终实际伤害。

## 玩家入伤结算顺序

玩家受到伤害时，`GameState.apply_damage(event)` 使用以下顺序：

1. 本地调试护身拦截。
2. `DamageCalculator.calculate()` 计算基础入伤。
3. 非 `pure` 伤害应用角色/被动的全局入伤倍率。
4. 生存共鸣 I/II 应用固定减伤和低血额外减伤。
5. 控制共鸣 II 对受控敌人的接触/弹体伤害应用威胁削弱。
6. 生存共鸣 III 若可触发或正在持续，则按本次未减免入伤转为治疗并记录吸收量。
7. 守卫折光甲胄按本次未减免入伤计算减免量，并将该减免量反射给附近敌人。
8. 扣血、治疗、记录受伤来源和死亡原因。

`pure` 伤害用于系统派生伤害，通常不吃普通入伤倍率和生存减伤。

## 效果分类

新增效果时先归入下面一种类型。

| 分类 | 定义 | 当前例子 | 推荐位置 |
|------|------|----------|----------|
| 伤害倍率 | 只改变本次伤害数值 | 弹幕远距增伤、近身贴近增伤、场地压制增伤、固定 +8% | `DamageCalculator.calculate()` 或后续的 modifier 层 |
| 目标状态 | 挂在敌人或玩家身上的持续状态 | `slow`、`stun`、`control_threat_reduction` | `StatusEffect` / 目标节点 |
| 目标特殊状态 | 需要目标本地字段或物理处理 | 场地压制层数、击退速度 | 目标节点；后续可迁移可叠层状态到 `StatusEffect` |
| 命中后效果 | 伤害成功后产生一次副作用 | 弹幕击退、控制充能、近身回放候选、场地叠层 | `DamageCalculator` 的 post-hit 分发 |
| 击杀后效果 | 只在击杀后触发 | 爆发过量外溢 | `DamageCalculator` 的 post-kill 分发 |
| 全局运行时效果 | 需要跨时间保存全局计时或蓄力 | 静滞能量、近身回放节奏、临界回响 | `GameState.run` 与 `GameState.add_run_time()` |
| 派生伤害 | 由系统效果再产生一次伤害 | 爆发外溢、近身回放、折光反射、临界回响反打 | 构造带标签的 `DamageEvent` |

## 当前状态清单

| 状态 | 类型 | 归属 | 说明 |
|------|------|------|------|
| `slow` | `StatusEffect` | 敌人 | 速度乘以 `value`，例如 `0.65` 表示 65% 移速 |
| `stun` | `StatusEffect` | 敌人 | 停止移动和攻击行为 |
| `control_threat_reduction` | `StatusEffect` | 敌人 | 玩家受到该敌人接触/弹体伤害时乘以状态值 |
| 场地压制 | 目标特殊状态 | 敌人字段 | 当前是 `_field_suppression_stacks` 和过期时间，后续适合迁移为可叠层 `StatusEffect` |
| 地脉封锁 | `slow` + 冷却字段 | 敌人字段 | 满层场地压制触发，冷却由 `_field_lockdown_next_trigger_time` 管理 |
| 击退 | 物理效果 | 敌人字段 | 使用 `_knockback_velocity/_knockback_timer`，不建议塞进普通状态 |

## 共鸣效果清单

| 共鸣 | I | II | III | 主要层级 |
|------|---|----|-----|----------|
| 近身 | 近身武器伤害 +8% | 近距离额外增伤 | 每 5 秒回放最近 5 秒最高近身伤害 | 伤害倍率 + 全局运行时效果 |
| 弹幕 | 弹幕武器伤害 +8% | 远距离额外增伤 | 命中击退 | 伤害倍率 + 命中后物理效果 |
| 场地 | 场地武器伤害 +8% | 命中叠压制并提高后续场地伤害 | 满层触发地脉封锁 | 伤害倍率 + 目标特殊状态 |
| 控制 | 控制武器伤害 +8% | 控制后降低目标威胁 | 控制充能满后全屏短眩晕 | 目标状态 + 全局蓄力 |
| 爆发 | 爆发武器伤害 +8% | 击杀时单体外溢 | 击杀时最多 3 目标外溢 | 击杀后派生伤害 |
| 生存 | 所受伤害降低 | 低血额外减伤 | 临界回响：伤害转治疗并反打 | 玩家入伤规则 + 全局运行时效果 |

## 派生伤害规则

派生伤害必须通过 `CombatEffectRules.add_tag_once()` 打明确标签，并在 `CombatEffectRules` 中集中声明是否允许继续触发其他效果。

| 标签 | 来源 | 当前规则 |
|------|------|----------|
| `burst_overflow` | 爆发共鸣 II/III | 不吃构筑共鸣倍率，不再次触发外溢，也不参与场地/控制/近身/弹幕副作用 |
| `melee_replay` | 近身共鸣 III | 不吃构筑共鸣倍率，不再次记录为回放候选，不计入武器统计 |
| `guardian_refraction` | 守卫折光甲胄 | 反射量基于未减免入伤计算，使用 `pure + reflect`，不计入武器统计 |
| `survival_echo` | 生存共鸣 III | 吸收量基于未减免入伤计算，使用 `pure + reflect`，不计入武器统计 |

新增派生伤害时必须：

- 使用明确标签。
- 在 `CombatEffectRules` 决定是否能触发构筑倍率、命中后效果、击杀后效果和战斗统计。
- 避免递归触发自身。

## 添加新效果的落点

| 需求 | 推荐做法 |
|------|----------|
| 只改变一次伤害数字 | 加到伤害倍率层，避免在目标里改 HP |
| 命中后附带减速/眩晕 | 如果伤害和状态绑定，用 `DamageEvent.status_id`；如果只是控制效果，用 `WeaponBase._apply_status_to()` |
| 状态需要刷新、延长、堆叠或 tick 伤害 | 用 `StatusEffect`，不要新增零散 timer 字段 |
| 状态影响敌人 AI 或移动 | 在敌人读取 `StatusEffect`，例如 slow/stun |
| 需要全局蓄力、冷却或窗口统计 | 放进 `GameState.run`，并在 `add_run_time()` 或明确事件点更新 |
| 击杀后连锁伤害 | 使用派生 `DamageEvent`，并加防递归标签 |
| 玩家受伤规则 | 放在 `GameState._calculate_incoming_damage()`，保持入伤顺序集中 |

## 建议重构顺序

当前不建议大改。推荐小步整理：

1. 保持现有玩法，先遵守本文档的分类和触发顺序。
2. 把 `DamageCalculator` 中的共鸣副作用拆出为独立的 post-hit/post-kill helper 文件或对象，但不改变外部 API。
3. 把场地压制迁移到 `StatusEffect` 的堆叠模型，减少敌人脚本里的专用状态字段。
4. 如新增派生伤害，继续扩展 `CombatEffectRules` 的统一规则表，集中声明哪些标签跳过倍率、统计、递归和副作用。
5. 再考虑 HUD/特效反馈，例如近身回放、临界回响和静滞脉冲的触发提示。
