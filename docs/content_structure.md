# 内容结构整理

> 本文档整理当前已经实现的角色、主动武器、局内被动强化和构筑方向。
> 长期武器蓝图见 [`weapon_system_blueprint.md`](./weapon_system_blueprint.md)，实现细节见 [`weapon_design.md`](./weapon_design.md)。

## 目标

当前已经有 20 种主动武器、10 种局内被动强化和 4 个默认角色。后续内容迭代的重点不应只是继续堆数量，而是让玩家能快速判断：

- 我这局适合走什么构筑。
- 一把新武器和已有武器有什么区别。
- 每个角色开局后自然想拿哪些武器和被动。
- 升级路径是在改变玩法，还是只是在堆数值。

本文档只描述当前实现，不包含召唤、商店、组合进化等尚未实现系统。

## 内容分层

| 层级 | 数量 | 作用 | 当前内容 |
|------|------|------|----------|
| 角色 | 4 | 决定开局武器、初始属性和局外身份 | 冒险者、游侠、守卫、炼金术士 |
| 主动武器 | 20 | 单局构筑核心，占武器槽，自动攻击或持续生效 | 6 槽上限，每把 3 条路径 |
| 局内被动强化 | 10 | 改变玩家或全局武器参数，占强化槽 | 6 槽上限，同类可升级 |
| 武器路径 | 60 | 单把武器内部的构筑分支 | 统一为强击、扩散、疾速、穿透、控制、守护、持续等路线主轴 |
| 敌人/Boss | 8 | 验证构筑是否成立的压力来源 | 小怪、远程、精英、Boss |

## 标签体系

当前标签拆成三层，避免把“武器形态”“路线数值”和“构筑奖励”混在一起。

| 层级 | 字段 | 用途 | 统一取值 |
|------|------|------|----------|
| 构筑标签 | `WeaponData.tags` | 计算构筑共鸣、推荐搭配和结算归类 | 近身、弹幕、场地、控制、爆发、生存 |
| 机制标签 | `WeaponData.mechanism_tags` | 说明武器差异，不直接作为构筑奖励门槛 | 斩击、弹体、环绕、陷阱、光束、连锁、反击、恢复、穿透、移动、持续等 |
| 路线标签 | `WeaponPath.build_tags` / `primary_build_tag` | 升级卡展示和路线主轴归类 | 强击、扩散、疾速、穿透、控制、守护、持续 |

`范围` 不再作为构筑标签使用：它过于宽泛，火场、环身、爆炸、光束都可能有范围。需要表达玩法时改用 `场地`、`近身`、`爆发` 或具体机制标签。

### 构筑共鸣 P1

第一阶段只使用 `WeaponData.tags` 计算共鸣，不把路线标签和机制标签纳入触发条件。共鸣奖励通过 `GameState.run.build_resonance_rewards` 记录，当前 6 个构筑标签均已接入实战效果。

| 规则 | 数值 |
|------|------|
| 武器第一个构筑标签基础分 | +1.0 |
| 武器第二个构筑标签基础分 | +0.5 |
| 选择流派专精 | 主标签 +0.5，副标签 +0.25 |
| 后续武器专精升级（Lv.3-Lv.8） | 主标签 +0.25/级，副标签 +0.125/级 |
| 局内被动专精 | 主标签 +0.2/级，副标签 +0.1/级 |
| 单把满级主标签专精 | 3.0 |
| 共鸣 I / II / III 阈值 | 2.0 / 4.0 / 6.0 |
| 展示位置 | 升级界面顶部共鸣条、选择卡标签、选择卡共鸣进度 |
| 奖励记录 | `GameState.run.build_resonance_rewards` |
| 近身 I | 近身武器伤害 +8% |
| 近身 II | 近身武器按命中距离反向增伤，160px 内开始，60px 内达到 +18% |
| 近身 III | 每 5.0s 回放最近 5.0s 内最高的一次近身武器伤害，命中 160px 内最近敌人 |
| 弹幕 I | 弹幕武器伤害 +8% |
| 弹幕 II | 弹幕武器按命中距离额外增伤，180px 后开始，420px 达到 +18% |
| 弹幕 III | 弹幕武器命中后击退敌人，精英 / Boss 击退力度降为 30% |
| 场地 I | 场地武器伤害 +8% |
| 场地 II | 场地命中叠加 2.0s 压制，每层后续场地伤害 +6%，最多 3 层 |
| 场地 III | 压制满层触发 1.2s 地脉封锁，普通敌人移速 -35%，精英 / Boss 移速 -15%；单敌人冷却 2.0s |
| 控制 I | 控制武器伤害 +8% |
| 控制 II | 控制武器施加减速 / 眩晕 / 击退后，使目标 2.0s 内接触和弹体伤害 -20% |
| 控制 III | 控制武器施加控制效果获得静滞能量，满 100 后自动全屏短眩晕；同一敌人充能冷却 0.5s |
| 爆发 I | 爆发武器伤害 +8% |
| 爆发 II | 爆发武器击杀时，将 80% 过量伤害外溢给 180px 内最近 1 个敌人 |
| 爆发 III | 爆发武器击杀时，将 100% 过量伤害外溢给 220px 内最近最多 3 个敌人；外溢不再次触发外溢 |
| 生存 I | 所受非纯粹伤害 -8% |
| 生存 II | 70% 生命以下额外减伤逐步提高，25% 生命以下达到额外 -22% |
| 生存 III | 濒死时触发 3.0s 临界回响，期间未减免伤害转治疗；结束后将吸收量分摊给 220px 内敌人，冷却 45s |

这个阶段的目标是让玩家能看到“我正在往哪个构筑走”。选择卡只保留标签和共鸣进度：例如 `近身 +0.25 1.75/2.0` 或 `近身 +1.0 解锁I`。不再额外展示推荐、偏离或路线解释文案；流派卡的路线方向由标签 chip 表达，专精贡献仍加到武器自身构筑标签上。局内被动作为辅助专精提供较小贡献，用来补足构筑而不是替代武器骨架。按当前 12 分钟局长估算，两把同构筑满级专精约可触发 III 档。后续再把路线标签作为修正分。

### 重复与边界

当前 20 把武器的覆盖面已经足够，但有几组容易在体验上混淆，需要后续通过数值、视觉和 UI 描述保持边界。

| 易混组 | 当前差异 | 后续应强化的边界 |
|--------|----------|------------------|
| 火焰瓶 / 毒液罐 / 火箭背包 | 都是持续场地 | 火焰瓶偏爆发投掷，毒液罐偏长时间覆盖，火箭背包偏移动路线和走位奖励 |
| 基础利刃 / 旋风斩 / 护盾球 | 都保护玩家近身 | 利刃偏方向操作，旋风斩偏周期清圈，护盾球偏被动防线 |
| 火焰瓶 / 毒液罐 / 地雷 / 锯片陷阱 | 都是场地部署或持续区域 | 火焰瓶偏爆发投掷，毒液罐偏长时间覆盖，地雷偏触发爆炸，锯片偏敌群定点切割 |
| 天雷引 / 地雷 / 火花弹 | 都是爆发范围伤害 | 天雷自动选敌，地雷预置陷阱，火花弹是弹体命中爆炸 |
| 弓箭精通 / 投掷斧 / 回旋镖 | 都是物理弹体 | 弓箭偏稳定高频，投掷斧偏高伤穿透，回旋镖偏往返覆盖 |
| 冰霜环 / 冲击波 | 都是环形控制 | 冰霜环偏减速和持续压制，冲击波偏瞬间打断和防线重置 |
| 激光笔 / 电磁链 | 都是中远距离多目标 | 激光笔偏几何直线，电磁链偏自动弹跳和索敌 |

如果未来要新增武器，优先填补新体验，而不是继续增加“环形范围伤害”“地面持续伤害”或“自动弹体”。

## 构筑原型

### 近身防线

| 项目 | 推荐内容 |
|------|----------|
| 核心武器 | 基础利刃、护盾球、旋风斩、荆棘护甲 |
| 补强武器 | 冰霜环、冲击波、回旋镖、圣光棱镜 |
| 推荐被动 | 生命强化、坚韧、生命源泉、扩张、专注 |
| 推荐角色 | 守卫、冒险者 |
| 体验目标 | 玩家可以贴近敌群走位，通过环绕、反击、减速和击退维持安全圈 |
| 风险 | 如果只堆防御，清怪效率不足，容易被远程敌人和 Boss 拖死 |

### 远程弹幕

| 项目 | 推荐内容 |
|------|----------|
| 核心武器 | 弓箭精通、散弹枪、投掷斧、激光笔 |
| 补强武器 | 回旋镖、电磁链、火花弹、天雷引 |
| 推荐被动 | 专注、强攻、扩张、疾风步、历练 |
| 推荐角色 | 游侠 |
| 体验目标 | 保持距离，通过弹体数量、穿透、连锁和射程在敌人靠近前清掉压力 |
| 风险 | 近身防线弱，需要至少一把冰霜环、冲击波、护盾球或旋风斩兜底 |

### 场地持续

| 项目 | 推荐内容 |
|------|----------|
| 核心武器 | 火焰瓶、毒液罐、地雷、锯片陷阱 |
| 补强武器 | 火箭背包、冰霜环、激光笔、圣光棱镜 |
| 推荐被动 | 余烬延续、扩张、专注、强攻、磁力增幅 |
| 推荐角色 | 炼金术士 |
| 体验目标 | 在地图上制造危险区域，让敌人追击路线经过火场、毒雾和陷阱 |
| 风险 | 前期即时清怪偏慢，需要爆发或控制补足开局压力 |

### 爆发清场

| 项目 | 推荐内容 |
|------|----------|
| 核心武器 | 天雷引、地雷、火花弹、散弹枪 |
| 补强武器 | 投掷斧、火焰瓶、圣光棱镜、冲击波 |
| 推荐被动 | 强攻、专注、扩张、历练 |
| 推荐角色 | 冒险者、游侠 |
| 体验目标 | 用低频高感知攻击处理怪群峰值，过量伤害外溢后适合点杀小队和处理精英旁杂兵 |
| 风险 | 冷却窗口明显，空窗期需要移动速度或控制武器支撑 |

### 控制风筝

| 项目 | 推荐内容 |
|------|----------|

| 补强武器 | 天雷引、激光笔、火箭背包、旋风斩 |
| 推荐被动 | 疾风步、扩张、专注、磁力增幅 |

### 生存续航

| 项目 | 推荐内容 |
|------|----------|
| 核心武器 | 圣光棱镜、荆棘护甲、护盾球、基础利刃 |
| 补强武器 | 冰霜环、冲击波、旋风斩 |
| 推荐被动 | 生命强化、生命源泉、坚韧、扩张 |
| 推荐角色 | 守卫、冒险者 |

## 角色构筑入口

| 角色 | 当前入口 | 自然构筑 | 需要补足 |
|------|----------|----------|----------|
| 冒险者 | 基础利刃 + 标准数值 | 近身防线、爆发清场、生存续航 | 根据第一二次升级决定方向；需要 UI 提示其“万能起手”定位 |
| 游侠 | 弓箭精通 + 高移速 + 弹体冷却 | 远程弹幕、控制风筝 | 至少补一把近身防线武器，否则被包围时容错偏低 |
| 守卫 | 荆棘护甲 + 高 HP + 折光甲胄 | 生存续航、近身防线 | 荆棘是受击触发，开局需要尽快拿护盾球、旋风斩、冰霜环或冲击波获得主动清怪能力 |
| 炼金术士 | 火焰瓶 + 范围/持续加成 | 场地持续、控制风筝 | 前期需要稳定单体或爆发武器，避免持续场地启动慢 |

角色自带被动应继续保持“职业入口”性质，不建议做成局内被动的复制版。例如角色被动可以改变起手节奏、风险偏好或武器选择倾向；局内被动则负责通用数值放大。

## 局内被动结构

| 被动 | 当前效果 | 主要服务对象 | 设计定位 |
|------|----------|--------------|----------|
| 疾风步 | 移动速度 +25 | 风筝、场地持续、拾取效率 | 操作空间和节奏 |
| 生命强化 | 最大 HP +30，当前 HP +30 | 近身、生存、守卫 | 容错上限 |
| 磁力增幅 | 拾取范围 +30 | 所有构筑，尤其场地持续 | 成长节奏 |
| 生命源泉 | 立即回血，之后定时回血 | 生存续航、近身防线 | 长线恢复 |
| 强攻 | 全武器伤害 +8% | 爆发、弹幕、近身 | 通用输出 |
| 专注 | 全武器冷却 -6% | 弹幕、爆发、控制 | 频率放大 |
| 扩张 | 全武器范围 +8% | 范围、控制、近身 | 空间放大 |
| 余烬延续 | 持续场地时间 +12% | 火焰瓶、毒液罐、火箭背包等 | 场地构筑专属放大 |
| 坚韧 | 受到伤害 -8% | 近身、生存、守卫 | 生存减伤 |
| 历练 | 经验获取 +10% | 所有构筑 | 成长加速 |

当前被动共鸣标签：

| 被动 | 共鸣标签 |
|------|----------|
| 疾风步 | 场地 / 生存 |
| 生命强化 | 生存 / 近身 |
| 磁力增幅 | 场地 |
| 生命源泉 | 生存 / 近身 |
| 强攻 | 爆发 / 近身 |
| 专注 | 弹幕 / 控制 |
| 扩张 | 场地 / 控制 |
| 余烬延续 | 场地 |
| 坚韧 | 生存 / 近身 |
| 历练 | 弹幕 |

当前被动池结构基本完整，但偏通用的被动较多。被动共鸣分用于表达“这个通用数值更适合服务哪个构筑”，后续如果要扩展被动，优先补足同等层级的专精被动，而不是继续加通用伤害、通用冷却。

可考虑的低复杂度专精被动方向：

| 方向 | 服务构筑 | 示例效果 |
|------|----------|----------|
| 弹体专精 | 远程弹幕 | 弹体速度或穿透小幅提升 |
| 控制专精 | 控制风筝 | 减速/眩晕持续时间小幅提升，但设上限 |
| 近身专精 | 近身防线 | 玩家附近武器范围或命中反馈增强 |
| 爆发专精 | 爆发清场 | 低频武器伤害提升，或冷却较长武器获得小补偿 |

## 武器路径结构

当前每把武器都有 3 条路径。整体结构已经从纯伤害堆叠调整为混合路径，但路径命名和构筑职责仍需要保持稳定。

| 武器 | 路径 A | 路径 B | 路径 C | 结构判断 |

## 当前内容问题

| 优先级 | 问题 | 影响 | 建议 |
|--------|------|------|------|
| P0 | 当前 20 把武器全部默认进池，早期信息量偏大 | 新玩家很难建立构筑预期 | 在不做解锁前，可先用角色推荐、升级卡标签和新手权重降低认知负担 |
| P0 | 守卫开局荆棘护甲主动性弱 | 玩家不主动受击时输出不足，受击又容易误判为应该硬吃伤害 | 提高守卫前几级拿到主动近身/控制武器的概率，或在角色 UI 明确推荐搭配 |
| P1 | 火焰瓶、毒液罐、火箭背包都依赖持续场地 | 如果视觉和描述不够清楚，玩家会觉得只是换皮 DOT | 强化三者的操作差异：投掷点、持久覆盖、移动尾焰 |
| P1 | 雷系武器存在自动索敌高收益重叠 | 天雷引和电磁链都可能让玩家少操作但高收益 | 天雷保持低频爆发，电磁链保持中频连锁，避免两者都同时扩大频率和目标数 |
| P1 | 通用被动过于稳定 | 玩家可能优先拿强攻/专注/扩张，专精构筑不明显 | 后续新增专精被动时控制通用被动权重或收益上限 |
| P2 | 武器路径 UI 只显示路线标签，不显示推荐联动 | 玩家知道“扩散/控制”，但不知道该配什么 | 后续可在卡片或详情面板显示推荐被动/推荐武器 |
| P2 | 结算页没有构筑复盘 | 难以判断本局内容结构是否成功 | 后续加武器伤害占比、升级路线和推荐构筑名 |

## 后续内容原则

新增内容时先问三个问题：

1. 它是否提供了当前 20 把武器没有的操作体验？
2. 它是否能自然归入某个构筑原型，或者创造一个新的低复杂度原型？
3. 它和已有角色、被动、路径是否有清晰联动？

短期不建议继续新增：

- 纯圆形范围伤害。
- 纯地面持续 DOT。
- 只改变颜色或伤害类型的弹体。
- 需要复杂召唤 AI、宠物寻路或多单位控制的武器。

短期更适合补：

- 构筑推荐表达：角色选择页、升级卡、暂停详情、结算页。
- 专精被动：弹体、控制、近身、爆发各补 1 个低复杂度被动。
- 数据采集：武器伤害占比和升级选择记录，用于验证构筑原型是否真的成立。
- 获取权重：根据角色和当前持有内容，提高推荐武器/被动出现概率，而不是一开始做完整解锁系统。
